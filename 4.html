The function has no elementary anti-derivative (i.e. one that can be written out as a combination of the usual functions). The value of the integral probably isn't elementary. We could just "call" the value \(c\), in the same way that Euler came up with the name \(e\), but that wouldn't be of any use. So, we're going to have to estimate the value numerically.
How do we estimate the value numerically? You've probably seen this procedure before. Divide up the interval \([0,4]\) into little sub-intervals, and then on top of each sub-interval draw a rectangle as high as the function is on the sub-interval. The combined area of these thin rectangles is pretty close to the value of the integral. And, the thinner we make all the rectangles to be, the closer we get to the actual value.
(We said to draw on each sub-interval a rectangle with the same height as the function on that sub-interval. You may ask: the function isn't quite constant on that sub-interval, so do we take the slightly shorter or slightly taller height? And how do we decide whether these tiny sub-intervals should be of equal width, or whether some can be wider than others? Riemann proved that for a continuous function like this, it doesn't matter. As long as we have some arrangement where every width is less than \(\epsilon\), and then push \(\epsilon\) towards zero, the combined area will always tend towards one fixed value.)
